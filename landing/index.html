Why ClearCache?

The browser already gives you several cache-clearing tools — and each is annoying in its own way.

Built-in option What it does What's annoying
Ctrl+Shift+R Reloads with cache bypass for that one navigation Doesn't unregister Service Workers — subsequent navigations may still be stale
DevTools → Disable cache Bypasses cache while DevTools is open You have to keep DevTools open. Doesn't help users testing your dev build.
Settings → Clear browsing data Wipes everything for the chosen time range Nukes every site's cache. Loses your auth sessions everywhere. Multi-step dialog.
ClearCache Clears this site's cache, unregisters its Service Workers, and hard-reloads Nothing — that's the point.

Features

Everything routed through one toolbar click, four keyboard shortcuts, and a right-click menu. No popup, no options page, nothing to configure.

🎯 Per-site by default

Click clears cache for the active tab's origin only. Other sites' data stays untouched.

⌨️ Three escalation modes

All-sites wipe, deep clear (cache + cookies + storage), and reload-all-tabs — each on its own keyboard shortcut.

✓ Toast + badge confirmation

A color-coded toast appears on the freshly reloaded page. Badge is the fallback on chrome:// pages.

📊 Cache hit/miss telemetry

The toast shows how many subresources were served from cache vs. fetched fresh, so you can verify the wipe worked.

🕶️ Reopen in incognito

Right-click for a one-click "deep-clear and reopen this URL in a fresh incognito window."

🌐 Internationalized

Available in English and Spanish. Drop a _locales/<lang>/messages.json to add more.

🪶 Tiny footprint

~150 lines of vanilla JS. No frameworks, no bundlers, no third-party dependencies.

🔒 Zero telemetry

Collects nothing, transmits nothing, contacts no remote servers. Fully auditable source.

All four modes

Defaults are sensible. Modifiers are explicit. No magic.

Action Shortcut Toast
Clear current site & reload (default) Alt+Shift+R green
Clear all sites & reload current tab Alt+Shift+A amber
Deep clear current site (cache + cookies + storage) Alt+Shift+D red
Reload all tabs in this window Alt+Shift+W green

Install

Store listings are coming soon. For now, side-load the latest GitHub Release. Every release publishes two zips — one for Chromium browsers, one for Firefox.

Chromium (Chrome, Edge, Brave, Arc, Opera, Vivaldi)

  1. Download clearcache-X.Y.Z.zip from the latest release.
  2. Verify the SHA-256 against the .sha256 file (optional but recommended).
  3. Extract the zip somewhere stable — the browser reads from this folder on every launch.
  4. Open chrome://extensions (or edge://extensions, brave://extensions, etc.).
  5. Toggle Developer mode on (top-right corner).
  6. Click Load unpacked and select the extracted folder.
  7. Pin the icon to your toolbar.

Firefox 112+

  1. Download clearcache-firefox-X.Y.Z.zip from the same release.
  2. Extract the zip somewhere stable.
  3. Open about:debugging#/runtime/this-firefox.
  4. Click Load Temporary Add-on and select any file inside the extracted folder.
  5. Pin the icon to your toolbar.
Firefox temporary add-ons are removed on browser restart. A permanent install will be available once the signed build lands on addons.mozilla.org.
Or build from source — there is no build step: 
git clone https://github.com/isaiasgv/ClearCache.git
Then load the cloned folder via Load unpacked (Chromium) or Load Temporary Add-on (Firefox).

Privacy

The extension collects nothing.

  • No analytics, no error reporting, no usage tracking.
  • No remote calls — the extension never makes an HTTP request.
  • No data storage. The extension reads no cookies, no localStorage, no IndexedDB.
  • No content scripts that read your pages.

Read the full privacy policy or audit the source — background.js is about 150 lines.

Open source & permissions

Every permission has a justification. There are no "just in case" requests.

PermissionWhy it's needed
browsingDataCalling chrome.browsingData.remove (or browser.browsingData.remove on Firefox) to clear cache, Cache Storage, Service Workers, and (deep mode) cookies / localStorage / IndexedDB.
tabsReading the active tab's URL to scope the clear, and reloading it.
contextMenusAdding the right-click entries on the toolbar icon.
scriptingInjecting the post-reload toast into the page. Toast injection is the only use.
webRequestRead-only observation of post-reload requests to compute cache hit/miss telemetry shown in the toast.
<all_urls>The per-site scoping logic must resolve any origin you visit. The extension never reads page contents.
